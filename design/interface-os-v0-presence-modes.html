Interface OS — Presence Modes

Four modes define how much of the surface the agent occupies. Each mode has distinct visual dimensions, agent control level, human interrupt affordances, and entry/exit conditions. All modes layer on top of the approved state machine grammar — they are not a second grammar.

Ambient
Background pulse only. Agent is present but silent.
Agent: watching
Minimal surface — pulse indicator only
Visual Dimensions
  • Single pulse dot (8px) in corner or edge of screen
  • No text, no cards, no stream
  • Colour encodes agent state: green=healthy, amber=waiting, red=error
Agent Control Level
  • Agent cannot expand or take focus autonomously in this mode
  • Agent may pulse faster to signal a waiting item
  • No UI mutations permitted beyond dot colour + pulse rate
Human Interrupt Affordances
  • Tap/click the dot → escalates to Conversational
  • Long-press → quick approve/deny if an approval is pending
  • No dismiss needed — ambient is the resting state
Trust / Consent Visibility
  • Trust badge: "Agent: watching" — always visible on dot hover/tap
  • No active consent prompt in ambient
  • Human is implicitly in control by not expanding
Entry Conditions
  • Default resting state after session collapse
  • Manual collapse from any higher mode
  • Inactivity timeout from Conversational (default: 60s)
Exit Conditions
  • Human taps dot → Conversational
  • Agent has pending approval → pulse signals, human taps → Conversational or quick-approve
  • Agent escalation request (requires human confirmation) → Conversational
Conversational
Soft surface with streaming text. Agent and human exchange.
Agent: active
Streaming text surface — partial screen
Visual Dimensions
  • Bottom sheet or side panel — max 40% of viewport
  • Live text stream from agent (token-by-token)
  • Human reply field always visible and focusable
  • Background content remains visible and interactive
Agent Control Level
  • Agent controls text content and streaming only
  • Agent cannot expand to Operational without explicit human action
  • Agent may suggest "open task view" — human must accept
Human Interrupt Affordances
  • Send reply → continue exchange
  • "Open task view" → Operational
  • Swipe down / tap X → collapse to Ambient
  • Stop stream button always visible during agent output
Trust / Consent Visibility
  • Trust badge in header: agent name + current action
  • Any real-world action proposed shows consent chip before execution
  • Human can deny any proposed action inline
Entry Conditions
  • Human taps ambient dot
  • Inbound message or notification accepted
  • Collapse from Operational via back/close
Exit Conditions
  • Human taps "Open task view" → Operational
  • Inactivity 60s → Ambient
  • Human collapses → Ambient
Operational
Focused task surface. Agent artifacts front and center.
Agent: working
📄 Draft: Email to client re: invoice
🔗 PR #902 — ready to merge
Visual Dimensions
  • Full-height panel or modal — 60–80% of viewport
  • Artifact cards: drafts, PRs, decisions, run status
  • Approve / Deny / Edit controls always visible for pending items
  • Background app visible but non-interactive by default
Agent Control Level
  • Agent updates artifact cards in real time from run stream
  • Agent cannot take external action without explicit approval
  • Agent may request Immersive for high-stakes tasks — human must accept
Human Interrupt Affordances
  • Approve / Deny / Edit on each artifact card
  • Back → Conversational
  • Collapse → Ambient
  • Stop run button always accessible in header
Trust / Consent Visibility
  • Every pending external action shows trust chip with: agent, action, target
  • Human must tap Approve before execution — no auto-approval
  • Deny is always one tap, no confirmation dialog
Entry Conditions
  • Human taps "Open task view" from Conversational
  • Approval item arrives and human accepts surface expansion
  • Collapse from Immersive
Exit Conditions
  • Human taps Back → Conversational
  • Human taps Collapse → Ambient
  • Agent requests Immersive, human accepts → Immersive
  • All artifacts resolved, inactivity 120s → Conversational
Immersive
Full-screen. Agent owns the surface for high-stakes tasks.
Agent surface active
Visual Dimensions
  • 100% viewport — agent owns everything
  • Background content not visible
  • Stop/Eject control always visible (cannot be hidden by agent)
  • High-contrast trust indicator always on screen
Agent Control Level
  • Agent controls full visual canvas
  • External actions still require explicit approval (cannot be waived in Immersive)
  • Agent cannot enter Immersive autonomously — human must accept the mode shift
Human Interrupt Affordances
  • Stop — halts current run immediately, collapses to Operational
  • Eject — force-exits to Ambient, ends session
  • Override chip — always accessible in top bar
  • Physical gesture (shake/volume) can also trigger Stop on mobile
Trust / Consent Visibility
  • Persistent trust bar at top: agent name, current action, elapsed time
  • Any external action shows full consent modal before execution
  • Cannot dismiss trust bar — it is always visible
  • Session consent was granted by human to enter this mode
Entry Conditions
  • Agent requests Immersive from Operational
  • Human reads request context and explicitly accepts
  • No other entry path — cannot auto-escalate
Exit Conditions
  • Human taps Stop → Operational
  • Human taps Eject → Ambient
  • Run completes → Operational (with result artifact)
  • Error / timeout → Operational (with error artifact)
Mode Transitions
Ambient→Conversational Human taps dot / notification tap
Conversational→Operational Human taps "Open task view"
Operational→Immersive Agent requests + human accepts
Immersive→Operational Stop / run complete / error
Operational→Conversational Human taps Back
Conversational→Ambient Human collapse / 60s inactivity
Any→Ambient Human Eject / explicit collapse
State machine layering: These 4 modes are a mode layer on top of the approved Interface OS v0 state machine (design/interface-os-v0-state-machine.html). They govern surface occupation and human control affordances within each state — they do not introduce a separate grammar or override state machine transitions.
